The Platform API uses **scopes** to define fine-grained access control. Scopes are selected when [creating an API key](/docs/admin/apikeys).

## How scopes work

Scope names follow a `resource:action` format (e.g. `devices:read`). 

Nearly all API endpoints require one or more scopes, shown on the endpoint's API Reference page. If an API key is missing a required scope, the request returns `403 Forbidden`.

## Scope tiers

Most resources follow a common tier pattern:

| Tier | Access |
|------|--------|
| `read` | Get or list operations |
| `write` | Create, update, or delete operations (varies by resource) |
| `delete` | Delete operations only (varies by resource) |
| `manage` | All operations—superset of all other scopes for the resource |


The `write` scope does **not** include `read` access—if you need both, select both scopes. 

For low-risk resources, the `write` scope includes `delete` access. However, for high-risk resources, such as devices, `delete` is a separate scope.

Some resources have additional granular scopes beyond the standard tiers (e.g. `deployments:stage`). The `manage` scope always includes everything for the resource.

## Scopes by resource

### Config instances

| Scope | Permissions |
|-------|-------------|
| `config_instances:read` | get, list |
| `config_instances:write` | create |
| `config_instances:manage` | all operations |

### Config schemas

| Scope | Permissions |

| `config_schemas:read` | get, list |
| `config_schemas:write` | create, update |

### Config types

| Scope | Permissions |
|-------|-------------|
| `config_types:read` | get, list |
| `config_types:write` | create, update |
| `config_types:manage` | all operations |

### Deployments

Deployments have a unique set of scopes due to the high risk of deploying to devices. 

The scopes are straightforward except for **remove** and **archive**:

- **remove** allows you to remove deployments which are <Tooltip tip={QUEUED_TOOLTIP.tip} cta={QUEUED_TOOLTIP.cta} href={QUEUED_TOOLTIP.href}>queued</Tooltip> or <Tooltip tip={DEPLOYED_TOOLTIP.tip} cta={DEPLOYED_TOOLTIP.cta} href={DEPLOYED_TOOLTIP.href}>deployed</Tooltip>
- **archive** allows you to archive deployments which are <Tooltip tip={STAGED_TOOLTIP.tip} cta={STAGED_TOOLTIP.cta} href={STAGED_TOOLTIP.href}>staged</Tooltip> or <Tooltip tip={DRIFTED_TOOLTIP.tip} cta={DRIFTED_TOOLTIP.cta} href={DRIFTED_TOOLTIP.href}>drifted</Tooltip>.


| Scope | Permissions |
|-------|-------------|
| `deployments:read` | get, list |
| `deployments:write` | create |
| `deployments:stage` | stage |
| `deployments:deploy` | deploy |
| `deployments:remove` | remove |
| `deployments:archive` | archive |
| `deployments:manage` | all operations |

## Principle of least privilege

While granting an API unrestricted access to your resources is convenient, it is a security risk. We recommend using the **principle of least privilege** to grant the least amount of access possible.
