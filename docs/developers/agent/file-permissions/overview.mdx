The Miru Agent runs as the `miru` system user and needs specific file system permissions
to operate. This section covers the agent's default permissions and how to grant it
access in the two cases that need it:

- [Configs](/developers/agent/file-permissions/configs) — the agent **writes** config
  instances to disk when it deploys a release.
- [Data uploads](/developers/agent/file-permissions/data-uploads) — the agent **reads**
  files on the device to upload them.

## Default permissions

During installation, the `miru` system user and group are created (no login shell). The
agent process runs as `miru:miru` under systemd.

By default, the agent creates and owns the following directories:

| Path | Purpose |
|------|---------|
| `/var/lib/miru` | Agent state, credentials, device identity |
| `/var/log/miru` | Log files (hourly rotation) |
| `/srv/miru` | Deployed configuration instance files |

The first two directories are only for internal use by the agent.

The third directory is the default location for deploying config instances—deploying to
any path inside `/srv/miru` is supported out-of-the-box, no additional configuration
required.
