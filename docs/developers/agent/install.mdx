This page details how to install the Miru Agent Debian package. However, the Miru Agent will not work until it has been activated as a device in your workspace. 

Once you have installed the `miru-agent` package, use the [device provisioning](/docs/learn/devices/provision/dashboard) documentation to activate the device.

## Upgrade

To upgrade a device's Miru Agent to a new version, you MUST 

1. Upgrade the `miru-agent` package
2. Reactivate the device

Upgrading the `miru-agent` package without reactivating the device may leave it in a stale or corrupted state. 

<Info>
    We are working to remove the need to reactivate the device after upgrading the `miru-agent` package. However, this is not yet supported.
</Info>

### 1. Package upgrade

As with the installation process, you can upgrade the `miru-agent` package via `apt` or manually by downloading and installing the new package. 

Since both methods use Debian packages, manual installations can be upgraded via `apt` and vice versa.

<Warning>
    You must [reactivate](#reactivation) the device after upgrading the `miru-agent` package. Otherwise, the device may be left in a stale or corrupted state.
</Warning>

<Tabs>
    <Tab title="APT">
    ```bash
    sudo apt-get update && sudo apt-get install --only-upgrade miru-agent
    ```

    <Info>
        The `--only-upgrade` flag ensures the `miru-cli` package isn't unintentionally upgraded.
    </Info>
    </Tab>
    <Tab title="Manual">
    Manual installations do not have a native upgrade mechanism. Simply repeat the [installation steps](#install) to upgrade to the latest version.
    </Tab>
</Tabs>

### 2. Device Reactivation

After upgrading the `miru-agent` package, you must reactivate the device to reset its state.

<Tabs>
    <Tab title="Dashboard">
        To upgrade a device from the dashboard, follow the [reactivation instructions](/docs/learn/devices/provision/dashboard#reactivate-a-device).
    </Tab>
    <Tab title="API Keys">
        To upgrade a device [via API Keys](/docs/learn/devices/provision/api-keys), rerun the provisioning process with the [`--allow-reactivation`](/docs/learn/devices/provision/api-keys#param-allow-reactivation) flag.
    </Tab>
</Tabs>
