## Upgrade

To upgrade a device's Miru Agent to a new version, you MUST 

1. Upgrade the `miru-agent` package
2. Reactivate the device

Upgrading the `miru-agent` package without reactivating the device may leave the device in a stale or corrupted state. 

<Info>
    We are working to remove the need to reactivate the device after upgrading the `miru-agent` package. However, this is not yet supported.
</Info>

### 1. Package upgrade

As with the installation process, you can upgrade the `miru-agent` package via `apt` or manually by downloading and installing the new package. 

Since both methods use Debian packages, manual installations can be upgraded via `apt` and vice versa.

<Tabs>
    <Tab title="APT">
        Upgrade the existing `miru-agent` via `apt`.

        ```bash
        sudo apt-get update
        sudo apt-get install miru-agent
        ```

        <Tip>
            Upgrade to a specific version with `sudo apt-get install miru-agent=<version>` (e.g. `0.7.0`, `0.8.0`, etc.).
        </Tip>
    </Tab>

    <Tab title="Manual">
        Manual installations do not have a native upgrade mechanism. Simply repeat the [installation steps](#install) to upgrade to the latest version.
    </Tab>
</Tabs>

### 2. Device Reactivation

After upgrading the `miru-agent` package, reactivate the device it's installed on.

**Dashboard**

To reactivate a device from the dashboard, visit the [reactivation section](/docs/learn/devices/manage#reactivate-a-device).

**API Keys**

To reactivate a device [via API Keys](/docs/learn/devices/provision/api-keys), simply execute the provisioning process with the [`--allow-reactivation`](/docs/learn/devices/provision/api-keys#param-allow-reactivation) flag.
