Miru uploads data *off* your devices and into *your* cloud storage.

You define a small set of [**upload rules**](/data-uploads/upload-rules) — what to
collect, when to collect it, and where it goes. The [Miru Agent](/developers/agent/overview)
detects matching data, confirms each file is complete, and uploads it directly to
your bucket.

## How it works

<img
  src="https://assets.mirurobotics.com/docs/v04/images/data-uploads/upload-flow.svg"
  alt="Upload flow: a device requests an upload from the Miru control plane, receives a short-lived signed URL for a path carrying its own device ID, and uploads the file directly to your cloud storage over HTTPS. Your data never passes through Miru."
/>

1. The device detects a completed file and makes an **upload request** to the Miru
   control plane.
2. The control plane verifies the device's identity and returns a **short-lived,
   single-use signed URL** for a path carrying that device's own ID.
3. The device performs a standard **HTTPS PUT**, uploading the file directly to your
   bucket.

Your data moves straight from the device to your storage — only the upload request
touches Miru.
