## Create an upload rule

Upload rules are created as part of a release: you define each rule as a YAML file in
your Git repository and pass it to `miru release create`. Visit
[Releasing upload rules](/data-uploads/releasing-upload-rules) to learn how.

Because rules are [deduplicated by digest](/data-uploads/upload-rules/overview#properties)
within their collection, releasing a rule identical to an existing one reuses that
rule rather than creating a duplicate.

## View an upload rule  {/* TODO: confirm required role/scope */}

Rules ship with releases, so a release lists the upload rules it contains alongside
its config schemas. Open a release and select an upload rule to inspect its
[source](/data-uploads/upload-rules/sources),
[destination](/data-uploads/upload-rules/destinations), digest, and
[collection](/data-uploads/upload-collections).

## Git commit metadata

Each rule records the Git commits it was released from: the commit itself and the rule
file's path relative to the repository root. If the same rule definition is released
again from a later commit, the existing rule accrues a link to that commit too — the
rule's commit history shows every commit that has shipped it.

## Editing and deleting rules

Upload rules **cannot be edited or deleted** — they are immutable by design. To change
what a rule collects or where it goes, edit the rule's YAML file and ship it in a new
[release](/fundamentals/releases/overview); this creates a new rule under the same
collection. The release a device is on determines which rules are active, so retiring
a rule is a matter of shipping a release that no longer includes it.
