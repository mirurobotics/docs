## Properties
<ParamField path="content" type="object">
  <ImmutableBadge />

  The configuration values that are deployed to the device as a JSON or YAML file.
</ParamField>

<ParamField path="file path" type="string">
  <ImmutableBadge />

  The absolute filesystem path where the config instance is written on the device.

  Examples: `/srv/miru/configs/v1/motion-control.json`, `/srv/miru/configs/safety.yaml`
</ParamField>

<ParamField path="config schema" type="Config Schema">
  <ImmutableBadge />

  The [config schema](/docs/learn/schemas/overview#properties) that the config instance must adhere to.

  Example: `SCH-FmoDN`
</ParamField>

## File formats

Currently, config instances support JSON and YAML. However, support for more formats, including XML, is coming soon!

Config instances support JSON and YAML. The Miru Agent parses both formats at deploy time and exposes the resulting data to on-device applications in the same way, so you can pick whichever format fits your workflow.
