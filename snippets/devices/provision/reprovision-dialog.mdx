[REPROVISION DIALOG]

The dialog will present a pre-authenticated command that invokes the `miru-agent` binary with a short-lived (5 minutes) token to reprovision the device for your workspace. The command looks like:

```bash
MIRU_PROVISIONING_TOKEN=<token> \
  /usr/sbin/miru-agent reprovision
```

- The first line sets the `MIRU_PROVISIONING_TOKEN` environment variable for the duration of the command.
- The second line invokes the `miru-agent` binary with the `reprovision` command.

To reprovision the device, copy the command from the dialog and paste it into the terminal of the machine you want to reprovision.

<Tip>
  For more information about Miru's authentication model, visit the [agent security](/docs/developers/agent/security) documentation.
</Tip>
