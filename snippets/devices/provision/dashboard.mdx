Click the **Activate** button on the device you just created.

<Frame>
  ![Miru UI screenshot](https://assets.mirurobotics.com/docs/v03/images/devices/one-device-to-activate.png)
</Frame>

A dialog will appear walking prompting you to install the `miru-agent`.

If the device already has the correct `miru-agent` package installed, you can skip the installation step. 

[INSTALLATION DIALOG]

<Note>
  For more information about installing the `miru-agent` package, visit the [agent installation](/docs/developers/agent/install) documentation.
</Note>

Once the `miru-agent` package is installed, continue to the activation step.

[PROVISIONING DIALOG]

The activation command invokes the `miru-agent` binary with a short-lived (5 minutes) provisioning token to activate the device for your workspace. The command will look similar to the following:

```bash
MIRU_PROVISIONING_TOKEN=<provisioning-token> \
  /usr/sbin/miru-agent provision
```

Copy the command from the dialog and paste it into a terminal of the device you want to provision.

<Tip>
  For more information about Miru's authentication model, visit the [agent security](/docs/developers/agent/security) documentation.
</Tip>
