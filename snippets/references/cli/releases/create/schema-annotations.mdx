<ParamField path="config type" required>
  The config type is a required annotation that identifies the config type to which a schema belongs. Below is the syntax for annotating a schema with a [config type slug](/docs/learn/config-types#param-slug).

  <CodeGroup>

  ```yaml JSON Schema
  x-miru-config-type: "planning"
  ```

  
  ```cue CUE
  @miru(config_type="planning")
  ```

  </CodeGroup>

  Examples: `mobility`, `communication`, `perception`
</ParamField>

<ParamField path="instance file path">
  The instance file path is the absolute filesystem path where config instances for this schema are written on the device.

  <Warning>
    The Miru Agent ships with out-of-the-box access to `/srv/miru`. To deploy configurations to a custom directory, please visit the [File Permissions](/docs/developers/agent/file-permissions) page.
  </Warning>

  Instance file paths control the type of file that config instances are deployed as. Currently, JSON (`.json`) and YAML (`.yaml`, `.yml`) are supported.

  This annotation is optional and defaults to `/srv/miru/configs/planning.json`.

  <CodeGroup>

  ```yaml JSON Schema
  x-miru-instance-filepath: "/srv/miru/configs/planning.json"
  ```

  
  ```cue CUE
  @miru(instance_filepath="/srv/miru/configs/planning.json")
  ```

  </CodeGroup>

  Examples: 

      - `/srv/miru/configs/mobility.json`
      - `/home/myapp/configs/communication.yaml`
      - `/var/lib/myapp/configs/safety.yaml`


</ParamField>
